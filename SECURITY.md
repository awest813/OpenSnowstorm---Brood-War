# Security Policy

## Supported versions

This project is in active development. Security fixes will generally land on the default branch.

## Reporting a vulnerability

If you believe you’ve found a security vulnerability:

- **Do not** open a public issue with exploit details.
- Prefer to report privately via GitHub Security Advisories (recommended).

If Security Advisories are not available for this repo, open a minimal issue that requests a private contact, without sensitive details.

## What to include

- A clear description of the issue and potential impact
- Steps to reproduce (or a proof-of-concept, if safe)
- Any suggested mitigations

## Disclosure process

Maintainers will:

- Confirm receipt and begin triage
- Work toward a fix and coordinate release notes if applicable
- Credit reporters when requested (and when appropriate)
